Architecture technique proposée (haut niveau)

Backend : Django (views, ORM, admin).

Asynchrone : Celery (worker), Broker : Redis (ou RabbitMQ).

Database : sqlite3 (conforme à la consigne).

IMAP : imaplib + email (pour parser) ou bibliothèque plus haut-niveau si tu veux.

Détection : implémentation de règles et fonctions heuristiques (Python). Possibilité d’ajouter librairies (virus scan, momentum).

Frontend : templates Django + JS léger pour dashboard (charts).

Logs : table IncidentLog + fichier de log.

APIs externes : requests.


Exemples de playbooks utiles à implémenter (idées)

Playbook Phishing élevé

Condition : incident_type == 'phishing' and score > 0.8

Actions : quarantine_email → notify_analyst → create_ticket → add_sender_to_watchlist

Playbook Spam massif

Condition : incident_type == 'spam' and score > 0.9

Actions : move_to_spam_folder → mark_as_read → log_action

Playbook pièce jointe suspecte

Condition : has_suspicious_attachment == True

Actions : save_attachment → send_to_antivirus_scan → notify_admin
